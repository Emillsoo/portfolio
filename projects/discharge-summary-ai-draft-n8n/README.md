# 입퇴원 요약 AI 초안 자동작성 (n8n · LLM)

> ⚠️ 공개본 안내: 실제 환자 임상데이터·EMR 인증정보·양식코드는 어떤 예시도 포함하지 않습니다. 데이터는 전부 합성입니다.

## 문제
전담간호팀·의료진이 입퇴원 요약과 입원초진기록을 임상기록 본문에서 수기로 요약 작성해야 했다.

## 접근
재원환자 임상기록을 AI가 항목별로 요약해 EMR에 **임시저장 초안**으로 적재한다. 최종 확정은 반드시 의사의 검토·전자서명(휴먼리뷰 게이트).

## 아키텍처
```
트리거(스케줄 / 챗 / executeWorkflow)
  → 대상자 조회(담당의 필터)
  → CLOB 본문 조각 조회(1,000자 × 12) → 병합
  → 항목별 LLM 5분담(초진/치료과정/검사결과/투약처방/추후계획)
  → Merge 동기화 게이트 → 8항목 병합
  → EMR 임시저장 적재(espiseq=0)
  → 성공판정(HTTP200 + ErrorCode0 + issuc=Y)
  → 감사로그(시트 11컬럼) / 실패 시 메일
  → (EMR 화면) 의사 검토·전자서명
```

## 핵심 기술 / 안전장치
- **CLOB 조각 분리 조회**로 대용량 본문 조회 제약 우회
- **다중 LLM 병렬 합류 동기화**: Merge `numberInputs` 게이트로 5분담 합류 보장
- 품질 규칙: 환각 금지·원문 충실, "[원문 없음]" → null 정규화
- **silent failure 방지**: 성공 판정을 다중 조건으로 검증
- 운영 안전: 실행 90분 제한, 테스트 서버 임포트 금지 패턴 문서화(스케줄 대량적재 방지)

## 역할
워크플로우 · 프롬프트 · 적재 연동 전담 개발.

## 보여주는 역량
의료 임상데이터에 대한 안전한 LLM 도입 — 품질통제·환각방지·인적검토 게이트 설계.

## 가상 재구현
`impl/` — 합성 임상노트에 대한 항목별 요약 파이프라인과 휴먼리뷰 게이트 데모.
