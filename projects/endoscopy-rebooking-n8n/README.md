# 내시경 예약 재배정 자동화 (n8n · EMR submit)

> ⚠️ 서브밋/테이블/화면경로/환자 예시는 포함하지 않습니다.

## 문제
의사 휴가 등으로 내시경 예약을 다른 슬롯/의사로 재배정해야 함. 자동화 v2가 "성공"으로 통보했으나 실제 DB는 미변경(silent failure)되는 사고 발생.

## 근본원인 분석 (핵심)
- EMR `setExecPrcpRsrv`의 `executeBatch`가 **요청 row 수를 반환(실 update 수 아님)** → 0 row여도 성공 응답 → silent failure로 규명.
- 운영 화면의 실제 호출 패턴 분석: `flag='rsrv'` + 환자 단위 다중 row 묶음, jparam 45필드 중 누락 필드 규명.

## 재설계(v3.0) / 아키텍처
```
대상 조회 → 화면 정합 호출패턴 구성 → 재배정 호출(.jnu)
  → DB 재조회 가드(호출 직후 실 변경 확인) → 성공 판정 → 모니터링 알림
```

## 핵심 안전장치
다층 가드(예외 + count + DB 재조회 + 합산), DEV 회귀/신규 케이스 검증 → active=false→수동1회→검증→active.

## 보여주는 역량
"응답 성공 ≠ 실제 처리"를 규명하고 **실데이터 검증**으로 신뢰성을 확보하는 디버깅·설계 역량.
