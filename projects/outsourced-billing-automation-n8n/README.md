# 위수탁검사비 수납처리 자동화 (n8n · EMR DML)

> ⚠️ 은행 계좌번호·서브밋ID·EMR URL·계정은 포함하지 않습니다. 수치 성과는 사용하지 않습니다.

## 문제
재단 병원 간 위·수탁검사비 미수금을 매월 통장입금으로 수기 수납 처리. 과거 중복키 버그로 오수납 장애 발생.

## 접근 / 아키텍처
```
스케줄/수동 → 전월 날짜계산 → 전월 미수 조회(EMR .jnu)
  → 대상 구성(가드) → 통장입금 수납(.jnu) → 결과 판정 → 알림
별도 모니터링 워크플로우: 수납 1시간 후 미처리 잔존 점검
```

## 핵심 안전장치 (금전 자동화)
- **미수코드 이중 필터**(조회 + 클라이언트 재필터) — 오수납 재발방지
- **부분수납 가드**: 잔액≠미수금 1건이라도 있으면 전체 abort — 과수납 방지
- **모니터링 동시 가동**: DB 재조회로 실처리 검증(성공응답≠실처리 대비)
- JsonReader 제약 대응(숫자→문자열, 빈배열 더미객체)

## 역할
워크플로우 전담 개발 · 장애 복구 · 재설계.

## 보여주는 역량
금전이 오가는 자동화의 안전장치(가드·모니터링·DB 재검증) 설계.
