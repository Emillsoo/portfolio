# 재원환자 불일치 자격확인 알림 자동화 (n8n · 데이터연계)

> ⚠️ 공개본 안내: 실제 [의료기관] EMR·환자정보·크리덴셜은 포함하지 않습니다. 화면/테이블/서브밋 구조는 일반화했고, 데이터는 모두 합성입니다.

## 문제
재원환자의 보험자격은 입원 중에도 변동한다. 담당자가 매일 EMR의 자격 불일치 화면을 눈으로 확인해야 했고, 놓치면 수납·청구 오류로 이어졌다.

## 접근
- 원내 자격과 공단(NHIC) 자격을 매일 자동 대조하고, 불일치 대상자만 시트·메일로 통보한다.
- **EMR 화면의 판단 로직을 SQL로 복제**해 "화면과 동일 기준"을 보장한다(보험유형·차상위·급여제한·체납).

## 아키텍처
```
매일 09:00 스케줄
  → (가드) 시트 정리/보존
  → 당일 기록 시트 생성
  → Oracle(READ-ONLY) 불일치 조회 (화면로직 SQL 복제)
  → 코드→한글 변환
  → 대상자 有: 시트 기록 + 건수 집계 + 메일 알림
     대상자 無: 정상 종료(0건 early-exit)
```

## 핵심 기술 / 안전장치
- 환자별 최신 공단 결과 1건 선택: `ROW_NUMBER() OVER (PARTITION BY 기관코드, 식별자 ORDER BY 조회시각 DESC)`
- **거짓 불일치 방지 3중 가드**: ① 공단 미수신 환자 비교 제외 ② 다기관 데이터 혼입 차단 ③ 차상위 변환 표준화
- **시트 보존 가드 3중**: 명명규칙 외 탭 보호 · 오늘 탭 보호 · 최소 1탭 보존
- **시간대 버그 해결**: `Asia/Seoul` 명시 적용으로 UTC 자정경계 날짜 어긋남 제거
- Fail-safe: 시트 정리 실패해도 알림 흐름 유지

## 역할
요구분석 · 화면로직 SQL 복제 · n8n 워크플로우 · 운영 안정화 전담.

## 보여주는 역량
현업 업무규칙의 데이터화, EMR 화면 비즈니스룰의 SQL 재현, 자동화 함정(시간대·중복·거짓양성) 방어와 운영 성숙도.

## 가상 재구현
`impl/` — 합성 환자/자격 데이터에 대해 동일한 대조 로직과 3중 가드를 재현한 데모(가상 병원 스키마).
