# 입원초진기록 초안 자동작성 (n8n · AI)

`n8n` · `LLM` · `Oracle` · `EMR 임시저장` · AI 활용

| 한 줄 | 재원환자 입원초진기록을 항목별 LLM으로 요약해 EMR 임시저장 초안으로 적재 |
|---|---|
| 역할 | 워크플로우·프롬프트·적재 연동 개발 (다기관 배포) |
| 핵심 역량 | 입퇴원 요약과 동일한 안전한 LLM 파이프라인을 다른 기록유형에 확장 적용 |
| 상태 | 운영 중 |

> ⚠️ 환자 임상데이터·EMR 인증정보 미포함(합성). 설계·안전장치는 [입퇴원 요약 AI 초안](../discharge-summary-ai-draft-n8n)과 공유.

## 개요
입퇴원 요약 AI 초안과 동계열 프로젝트. 입원 시점의 초진기록을 대상으로, 임상기록 본문을 항목별 LLM이 요약해 EMR에 **임시저장 초안**으로 적재하고, **의사 검토·전자서명 게이트**를 둔다. 부천/인천 등 다기관에 배포.

## 공유 설계 (상세는 입퇴원 요약 프로젝트 참조)
- CLOB 조각조회 → 병합
- 항목별 LLM 분담 → Merge 동기화 게이트
- 품질규칙(환각 방지, 원문 없음 → null)
- 적재 성공판정 3중조건(HTTP200 + ErrorCode0 + issuc=Y) — silent failure 대비
- 감사로그 + 실패 알림, 휴먼리뷰 게이트

## 차별점
기록유형(입원초진)에 맞춘 항목 구성과 프롬프트, 다기관 동시 배포·모니터링 운영.
